6.1 HOW LOGIN WORKS (Owner-controlled OTP system)
--------------------------------------------------------------------------------

  Step 1: Owner goes to "Add Employee" in the app
  Step 2: Fills employee name + role
  Step 3: App generates a unique code (6-8 characters), valid for 5 MINUTES
  Step 4: Code shown to owner only
  Step 5: Owner verbally gives code to employee
  Step 6: Employee opens app → "Login with Code" → enters code
  Step 7: If valid and not expired → employee is logged in and stays logged in
  Step 8: Code is marked as used, cannot be reused

  OWNER ACCESS: Owner's account is pre-configured during app setup by developer.
  Owner has a separate, never-expiring secure passcode set at app initialization.

9. MEDIA HANDLING — PHOTOS & VOICE RECORDINGS
================================================================================

--------------------------------------------------------------------------------
9.1 HOW PHOTO CAPTURE WORKS (In Browser)
--------------------------------------------------------------------------------

  <input type="file" accept="image/*" capture="environment">

  - On mobile: opens camera directly
  - On desktop: opens file picker
  - Photo stored as File/Blob → uploaded to storage before sending form

--------------------------------------------------------------------------------
9.2 HOW VOICE RECORDING WORKS (In Browser)
--------------------------------------------------------------------------------

  - MediaRecorder API: navigator.mediaDevices.getUserMedia({ audio: true })
  - Browser asks for microphone permission (one-time prompt)
  - User taps Record → speaks → taps Stop → audio stored as Blob (.webm)
  - Blob uploaded to storage before form submission
  - REQUIRES HTTPS (Firebase Hosting and GitHub Pages both support HTTPS ✓)

--------------------------------------------------------------------------------
9.3 FILE STORAGE PATH STRUCTURE
--------------------------------------------------------------------------------

  /enquiries/breakdown/[enquiry-id]/photo.jpg
  /enquiries/breakdown/[enquiry-id]/voice.webm
  /enquiries/part-enquiry/[enquiry-id]/photo.jpg
  /enquiries/part-enquiry/[enquiry-id]/voice.webm
  /enquiries/other-service/[enquiry-id]/photo.jpg
  /enquiries/other-service/[enquiry-id]/voice.webm
  /products/[product-id]/image-1.jpg
  /products/[product-id]/image-2.jpg

  Download URLs saved in Firestore document alongside enquiry data.

--------------------------------------------------------------------------------
10.1 STORAGE COST COMPARISON (IMPORTANT — READ THIS)
--------------------------------------------------------------------------------

The user asked: "Does Firebase Storage require a subscription? What about
when orders increase and many photos/audio files pile up?"

ANSWER: YES, at scale, storage costs money. Here is a breakdown:

--- OPTION A: Firebase Storage (Google Cloud) ---

  Free tier: 5 GB storage, 1 GB/day download (requires Blaze plan since 2024)
  Cost after free: ~$0.026 per GB per month for storage
               + ~$0.12 per GB for download/egress

  Example estimate for Nexus Machinery:
    - 500 service enquiries/month, avg 2 MB per enquiry (photo + voice)
    - Storage needed: 500 × 2 MB = 1 GB per month → ~$0.026/month
    - 200 GB total after 3 years → ~$5/month storage
    - Download (executives viewing media): 50 GB/month → ~$6/month
    - TOTAL estimate: ~$10–15/month at scale

  VERDICT: Affordable for this business scale. Firebase is already integrated
  into the project (adminsdk JSON exists). RECOMMENDED for Phase 1.

--- OPTION B: Cloudinary ---

  Free tier: 25 GB storage + 25 GB bandwidth per month
  Paid: Credit-based ($89+/month for paid plan)
  BEST FOR: If you need automatic image optimization, resizing, CDN delivery.
  VERDICT: Overkill for this use case. Too expensive.

--- OPTION C: Backblaze B2 ---

  Free tier: 10 GB storage, 1 GB/day free download
  Cost: $0.006 per GB storage (4x CHEAPER than Firebase)
        $0.01 per GB download (but FREE if using Cloudflare CDN)

  Example estimate for Nexus Machinery at same scale:
    - 200 GB storage after 3 years → $1.20/month storage
    - Download via Cloudflare → FREE
    - TOTAL estimate: ~$1–2/month at scale

  VERDICT: Much cheaper than Firebase for large volumes. But requires
  separate setup (not integrated with Firebase Auth/Firestore). Extra work.
  RECOMMENDED for Phase 2+ if storage becomes a concern.

--- OUR RECOMMENDATION ---

  Phase 1: Use Firebase Storage (already set up, easy integration, free tier
  covers initial months). Monitor usage for first 6 months.

  Phase 2+: If enquiries grow significantly (hundreds per day), evaluate
  migrating to Backblaze B2 (save ~80% on storage costs). This migration
  is straightforward since Backblaze B2 supports AWS S3-compatible API.

  BOTTOM LINE: For the first 1-2 years at Nexus Machinery's expected volume,
  Firebase free tier + minimal pay-as-you-go will cost effectively ZERO.
  Even at full scale, it should be under ₹1,500/month. Not a concern right now.
